\chapter*{Abstract}

Automation and autonomous robotic systems are becoming increasingly important in modern industries and smart service sectors. Among these systems, autonomous line following robots are widely used in industrial transportation, warehouse management, hospital services, and restaurant automation.

This thesis presents the design and implementation of an autonomous line following food delivery robot for smart restaurant services using a 5-channel infrared sensor array and Arduino Nano microcontroller. The proposed system is capable of following predefined paths autonomously while maintaining stable movement through real-time sensor feedback and PD-based error correction.

The robot utilizes infrared sensors to detect black lines on a white surface. Based on sensor readings, the control algorithm continuously calculates positional errors and adjusts motor speed accordingly using proportional-derivative control logic. The hardware architecture consists of an Arduino Nano, L298N motor driver, N20 geared motors, lithium-ion battery system, and lightweight chassis.

Experimental results demonstrate that the robot can efficiently navigate both straight and curved paths with improved stability and fast response time. The developed system offers a low-cost and energy-efficient solution suitable for smart restaurant automation and indoor autonomous delivery applications.

\chapter{Introduction}

\section{Background}

Autonomous robotic systems are rapidly transforming modern industries and service environments. These systems reduce human effort, improve operational efficiency, and enhance productivity through automated task execution. One of the most widely studied autonomous systems is the line following robot.

A line following robot is an autonomous vehicle capable of detecting and following a predefined path using sensing technologies. These robots are commonly used in warehouse automation, industrial transportation, healthcare services, and smart restaurant systems.

The increasing demand for contactless services and smart automation has created the need for autonomous food delivery systems in restaurants. Traditional food serving methods require human labor and may reduce operational efficiency during busy hours. Therefore, autonomous restaurant delivery robots can significantly improve service quality and reduce workload.

This thesis focuses on the design and implementation of a low-cost autonomous food delivery robot using line following technology and PD-based control algorithms.

%               PROBLEM STATEMENT

Traditional restaurant food delivery systems rely heavily on human waiters and manual transportation methods. During peak operational periods, these methods may become inefficient, time-consuming, and prone to human error.

Existing low-cost autonomous robots often suffer from poor stability, inaccurate line tracking, slow response time, and weak recovery mechanisms. Therefore, there is a need for a stable, efficient, and cost-effective autonomous food delivery robot capable of accurate indoor navigation.

\chapter{Proposed System}

\section{Overview of the Proposed System}

The proposed system is an autonomous line following food delivery robot designed for smart restaurant environments. The robot is capable of navigating predefined indoor paths autonomously using infrared sensing technology and real-time control algorithms.

The primary objective of the system is to reduce human effort in restaurant food transportation by implementing an intelligent indoor delivery mechanism. The robot follows a black line on a white surface using a 5-channel infrared sensor array and continuously adjusts its movement using a proportional-derivative (PD) control algorithm.

The complete system consists of the following major subsystems:

\begin{itemize}
\item Sensor Subsystem
\item Control Unit
\item Motor Driver Circuit
\item Locomotion System
\item Power Supply System
\item Mechanical Chassis
\end{itemize}

The robot continuously detects the line position using IR sensors and computes the positional error relative to the center position. Based on the calculated error, the control system dynamically adjusts the speed of the left and right motors to maintain stable navigation.

\section{Working Principle}

The working principle of the robot is based on real-time line detection and dynamic motor speed correction.

Initially, the IR sensor array continuously scans the surface beneath the robot. When the sensors detect the black line, the reflected infrared intensity decreases. The sensor readings are analyzed by the Arduino Nano to determine the relative position of the line.

If the robot deviates from the center position, the control algorithm calculates the positional error and generates correction values. The left and right motor speeds are adjusted dynamically to steer the robot back toward the line.

The robot also contains an autonomous recovery mechanism. If the line is temporarily lost, the robot attempts to recover the line based on the previously detected direction.

% =====================================================

\section{Navigation Mechanism}

The navigation system of the robot is based on differential drive motion control. The turning mechanism is achieved by varying the rotational speeds of the left and right motors.

The navigation process can be summarized as follows:

\begin{enumerate}
\item Detect line position using IR sensors
\item Calculate positional error
\item Apply PD correction
\item Adjust motor speed dynamically
\item Maintain stable line tracking
\item Recover line if detection fails
\end{enumerate}

The robot continuously repeats this process in real-time during operation.

\section{Line Detection Method}

The robot identifies the black line by comparing sensor readings against a predefined threshold value.

If the sensor reading becomes lower than the threshold value, the corresponding sensor is considered to be detecting the black line.

The line position is determined by calculating the weighted average of active sensor positions.

The weighted position calculation improves navigation accuracy and enables smoother turning performance.

\section{Autonomous Line Recovery Mechanism}

During operation, the robot may temporarily lose the navigation line due to sharp turns or sensor misalignment.

If no sensor detects the line, the robot attempts to recover the line by rotating toward the previously detected direction.

The robot stores the last known direction of the line and uses that information for recovery.

This mechanism significantly improves navigation reliability and reduces path loss failure.

#define IN1 11
#define IN2 13
#define ENA 5
#define IN3 12
#define IN4 10
#define ENB 3
#define num_sensors 5

int sensor_pins[num_sensors] = {A0, A1, A2, A3, A4};
int sValues[num_sensors];
int base_speed = 150;
int max_speed = 220;
int turn_speed = 90;
float kp = 0.08;
float kd = 0.2;
int prev_error = 0;
int last_side = 0;
int center_offset = 0;
bool calibrated = false;

void setup() {
  pinMode(IN1, OUTPUT); pinMode(IN2, OUTPUT); pinMode(ENA, OUTPUT);
  pinMode(IN3, OUTPUT); pinMode(IN4, OUTPUT); pinMode(ENB, OUTPUT);
  for (int i = 0; i < num_sensors; i++) {
    pinMode(sensor_pins[i], INPUT);
  }
  Serial.begin(9600);
  
  calibrate_center();
}

void calibrate_center() {
  long sum = 0;
  int count = 0;
  
  for(int t=0; t<50; t++) {
    long temp_sum = 0;
    long temp_weight = 0;
    int temp_count = 0;
    
    for (int i = 0; i < num_sensors; i++) {
      int val = analogRead(sensor_pins[i]);
      if (val < 450) {
        temp_sum += (i * 1000);
        temp_weight += 1000;
        temp_count++;
      }
    }
    
    if(temp_count > 0) {
      int pos = temp_sum / temp_weight;
      sum += pos;
      count++;
    }
    delay(5);
  }
  
  if(count > 0) {
    int avg_pos = sum / count;
    center_offset = avg_pos - 2000;
  }
  
  calibrated = true;
}

void loop() {
  if(calibrated) {
    line_follow();
  }
}

int read_sensors() {
  long sum = 0;
  long weight = 0;
  int count = 0;
  
  for (int i = 0; i < num_sensors; i++) {
    int val = analogRead(sensor_pins[i]);
    sValues[i] = val;
    
    if (val < 450) {
      sum += (i * 1000);
      weight += 1000;
      count++;
    }
  }
  
  if (count == 0) return -999;
  if (count == 5) return 9999;
  
  if (sValues[0] < 450) last_side = -1;
  if (sValues[4] < 450) last_side = 1;
  
  int linePos = sum / weight;
  return (linePos - 2000 - center_offset);
}

void line_follow() {
  int error = read_sensors();
  
  if (error == -999) {
    if (last_side == -1) {
      motor_drive(-turn_speed, turn_speed);
    } else {
      motor_drive(turn_speed, -turn_speed);
    }
    return;
  }
  
  if (error == 9999) {
    motor_drive(base_speed, base_speed);
    return;
  }
  
  int adjusted_error = error;
  if(abs(error) < 200) {
    adjusted_error = 0;
  } else if(abs(error) < 400) {
    adjusted_error = error * 0.5;
  }
  
  float P = adjusted_error;
  float D = (adjusted_error - prev_error) * 0.6;
  int adjustment = (int)((kp * P) + (kd * D));
  prev_error = adjusted_error;
  
  adjustment = constrain(adjustment, -50, 50);
  
  int left = base_speed + adjustment;
  int right = base_speed - adjustment;
  
  left = constrain(left, 80, max_speed);
  right = constrain(right, 80, max_speed);
  
  motor_drive(left, right);
}

void motor_drive(int leftSpeed, int rightSpeed) {
  leftSpeed = constrain(leftSpeed, -max_speed, max_speed);
  rightSpeed = constrain(rightSpeed, -max_speed, max_speed);
  
  if (leftSpeed >= 0) {
    digitalWrite(IN1, LOW);
    digitalWrite(IN2, HIGH);
  } else {
    digitalWrite(IN1, HIGH);
    digitalWrite(IN2, LOW);
  }
  analogWrite(ENA, abs(leftSpeed));
  
  if (rightSpeed >= 0) {
    digitalWrite(IN3, HIGH);
    digitalWrite(IN4, LOW);
  } else {
    digitalWrite(IN3, LOW);
    digitalWrite(IN4, HIGH);
  }
  analogWrite(ENB, abs(rightSpeed));
}

\chapter{Challenges and Limitations}

\section{Introduction}

This chapter discusses the major challenges encountered during the design, implementation, and testing phases of the proposed autonomous line following food delivery robot.

Several technical and mechanical difficulties were identified throughout the development process. These challenges influenced navigation stability, sensor accuracy, motor performance, and overall system reliability.

% =====================================================
%         SENSOR RELATED CHALLENGES
% =====================================================

\section{Sensor Related Challenges}

The infrared sensor array was highly sensitive to environmental lighting conditions and surface reflections.

During experimentation, excessive ambient light occasionally affected sensor readings and reduced line detection accuracy.

Different surface textures also produced inconsistent infrared reflections, resulting in sensor noise and unstable navigation under certain conditions.

% =====================================================
%         MOTOR RELATED CHALLENGES
% =====================================================

\section{Motor Synchronization Issues}

The robot uses two N20 geared motors for differential drive movement.

Although both motors were identical in specification, slight variations in rotational speed were observed during operation. These speed mismatches occasionally caused directional deviation and unstable movement.

Repeated calibration and PWM adjustment were required to minimize synchronization problems.

% =====================================================
%         POWER RELATED CHALLENGES
% =====================================================

\section{Power Supply Fluctuation}

Battery voltage fluctuation was another major challenge during implementation.

As the battery charge decreased, the motor torque and response speed were affected. This occasionally reduced navigation stability during sharp turns and continuous movement.

Stable power management was necessary to maintain consistent robot performance.

% =====================================================
%         MECHANICAL CHALLENGES
% =====================================================

\section{Mechanical Structure Challenges}

Several mechanical implementation challenges were encountered during chassis construction and component mounting.

These challenges included:

\begin{itemize}

\item Sensor alignment issues

\item Wheel balancing problems

\item Chassis vibration

\item Loose wiring connections

\item Weight distribution imbalance

\end{itemize}

Proper component positioning and repeated structural adjustments were required to improve robot stability.

% =====================================================
%         PD TUNING CHALLENGES
% =====================================================

\section{PD Controller Tuning}

Selecting suitable PD control parameters was one of the most critical challenges.

Improper proportional gain caused excessive oscillation, while incorrect derivative gain reduced response speed.

Multiple experimental tests were conducted to determine stable \(K_p\) and \(K_d\) values for efficient navigation.

% =====================================================
%         SHARP TURN CHALLENGES
% =====================================================

\section{Sharp Turn Navigation Challenges}

The robot occasionally experienced instability during sharp turns at high speed.

Fast directional changes increased positional error and temporarily affected sensor detection accuracy.

Although the PD controller significantly improved turning performance, minor oscillations still occurred during aggressive turning conditions.

% =====================================================
%         RECOVERY CHALLENGES
% =====================================================

\section{Line Recovery Challenges}

The autonomous recovery mechanism improved navigation reliability, but recovery delays were occasionally observed when the robot lost the line completely.

Complex track conditions sometimes required additional recovery time before stable navigation could resume.

% =====================================================
%         SOFTWARE CHALLENGES
% =====================================================

\section{Software Implementation Challenges}

Several software-related challenges were also encountered during implementation.

These included:

\begin{itemize}

\item Sensor threshold tuning

\item Real-time data processing

\item PWM optimization

\item Error filtering

\item Noise reduction

\end{itemize}

Continuous debugging and software optimization were necessary to improve system performance.

\chapter{Conclusion}

\section{Conclusion}

This thesis presented the design and implementation of an autonomous line following food delivery robot for smart restaurant services using a 5-channel infrared sensor array and Arduino Nano microcontroller.

The proposed system successfully demonstrated stable autonomous navigation through real-time line detection and proportional-derivative (PD) control. The robot was capable of following predefined paths accurately while maintaining smooth directional correction and efficient turning performance.

The integration of a 5-channel IR sensor array improved line detection accuracy, while the PD control algorithm significantly reduced oscillation and enhanced movement stability.

Experimental results confirmed that the robot could efficiently navigate straight paths, curved tracks, and sharp turns with reliable line recovery capability.

The proposed robotic system offers several advantages including:

\begin{itemize}

\item Low implementation cost

\item Compact mechanical design

\item Stable autonomous navigation

\item Efficient line recovery mechanism

\item Real-time control response

\item Low power consumption

\end{itemize}

The developed system demonstrates the practical feasibility of low-cost autonomous food delivery robots for smart restaurant automation and indoor transportation applications.

Although several limitations remain, the proposed system provides a strong foundation for future intelligent robotic research and development.

Future integration of artificial intelligence, obstacle avoidance, IoT communication, and computer vision technologies may significantly improve the capability and intelligence of the robotic platform.
